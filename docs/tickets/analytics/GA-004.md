# GA-004 — Operationalize podcast GA4 reporting

- **Status:** in-progress
- **Priority:** P1
- **Topic:** Analytics

## Problem

Podcast interaction tracking now exists in code, but the GA4 side still needs explicit configuration and reporting conventions so the data becomes usable.

## Goal

Make podcast analytics actionable in GA4 for traffic, plays, completions, progress, and downloads.

## Proposed Work

- Register and validate podcast event reporting for:
  - `podcast_play`
  - `podcast_progress`
  - `podcast_complete`
  - `podcast_download`
- Confirm `/podcast` traffic is read from `page_view` using `page_path = /podcast`.
- Create/verify the GA4 custom dimensions needed for:
  - `podcast_slug`
  - `podcast_title`
  - `podcast_category`
  - `podcast_episode_number`
  - `language`
  - `source`
- Validate events in DebugView and standardize how the team reads:
  - most played podcast
  - most completed podcast
  - most downloaded podcast

## Notes

- Code instrumentation already exists in `src/components/AudioPlayer.tsx` and `src/analytics/events.ts`.
- This ticket is now mostly about GA4 administration, validation, and reporting hygiene.
